2 Configure Your Virtual Machine (8 min)

Now for the fun part - designing your cloud computer! Each choice affects performance and cost.

üìä Step-by-Step VM Configuration:
üíª 1. Basics Tab
  • Subscription: Azure subscription 1 (should be auto-selected)
  • Resource group: Create new ‚Üí "MyFirstVM-RG"
  • Virtual machine name: "MyFirstVM" (or your choice)
  • Region: Pick closest to you for best performance
  • Image: Windows Server 2022 or Ubuntu 20.04 LTS
  • Size: Standard_B1s (cheapest) or Standard_B2s (better)
üîë 2. Authentication
  • Username: azureuser (or your choice, avoid "admin")
  • Password: Complex password (12+ chars, mixed case, numbers, symbols)
  • Confirm password: Same as above
  • Public inbound ports: Allow RDP (3389) for Windows or SSH (22) for Linux
üí∞ 3. Cost Estimation
  • Standard_B1s: ~$8-15/month (perfect for learning)
  • Standard_B2s: ~$30-40/month (better performance)
  • Remember: You have $200 free credit!
  • Turn off when not using to save money
üí° Beginner Tips:
  • Start small - you can always resize later
  • Choose Windows if you want familiar interface
  • Choose Linux if you want to learn command line
  • Write down your username/password somewhere safe!

üîß Troubleshooting: When Things Don't Go as Planned

Don't panic! Every Azure explorer faces challenges. Here are the most common issues and how to fix them:

üö® Common Account Setup Issues
üö´ Common Problems & Solutions:
üìû Phone Verification Failed

Solutions:

  • Try "Call me" instead of "Text me"
  • Ensure your number includes the correct country code
  • Wait 10 minutes and try again
  • Use a different phone number
  • Check if your carrier blocks international texts
üí≥ Credit Card Rejected

Solutions:

  • Contact your bank - they might be blocking international transactions
  • Try a different card (debit often works better than credit)
  • Ensure billing address matches your card exactly
  • Wait 24 hours and try again
  • Try using a prepaid Visa/Mastercard
üìß Email Already in Use

Solutions:

  • You might already have a Microsoft account - try signing in
  • Check if you have an old Outlook/Xbox/Office account
  • Use a different email address
  • Try the account recovery option
‚ö†Ô∏è VM Creation Problems
üö´ VM Creation Issues:
üíµ "Quota Exceeded" Error

What it means: You've hit a limit on your free account

Solutions:

  • Choose a smaller VM size (Standard_B1s is the smallest)
  • Try a different region (some regions have more capacity)
  • Delete old VMs you're not using
  • Wait a few hours and try again
üíª "VM Size Not Available"

Solutions:

  • Try a different region (East US, West Europe usually have good availability)
  • Choose a more common VM size (B1s, B2s, D2s)
  • Wait a few hours and try again
  • Check Azure status page for known issues
üîë "Authentication Failed"

Solutions:

  • Make sure password meets complexity requirements (12+ chars, mixed case, numbers, symbols)
  • Username can't be "admin", "administrator", "root", or "user"
  • Try a completely different username/password combination
  • Use SSH key authentication instead (for Linux VMs)
üåê "Deployment Failed"

Solutions:

  • Click on the error details for specific information
  • Try a different VM size or region
  • Check if you have enough free credits remaining
  • Delete the failed resource group and start fresh
